What is a Stack?

// The fundamental LIFO data structure — from zero

🧠 Analogy — Stack of Plates

Imagine plates stacked in a cafeteria. You always put new plates on top, and you always take from the top. You can NEVER take a plate from the middle without removing the ones above it first. That's exactly a stack.

LIFO — Last In, First Out

LIFO = the last element you inserted is the FIRST to come out. The opposite of a queue.

STACK STATE
10 (pushed first)
20
30 ← TOP
⟹

When we call pop(), element 30 comes out first — even though 10 was pushed first.

Push order: 10 → 20 → 30
Pop order: 30 → 20 → 10

Core Operations

PUSH ⬆️

Add an element to the top of the stack.

POP ⬇️

Remove and return the top element.

PEEK / TOP 👁️

View the top element without removing it.

isEmpty()

Returns true if the stack has no elements.

isFull()

Returns true if array stack is at capacity.

size()

Returns the count of current elements.

Real World Uses

Browser Back Button

Every page you visit is pushed. When you press Back, the current page is popped and you return to the previous one.

Undo / Redo

Every action is pushed onto an undo stack. Ctrl+Z pops the last action and reverses it.

Function Call Stack

When main() calls foo(), the return address is pushed. When foo() returns, it pops back.

Expression Evaluation

Compilers use stacks to evaluate arithmetic. Covered in Tab 05!

Big O Complexity

OperationTimeWhy?
push()O(1)Always add to top — no searching
pop()O(1)Always remove from top — direct access
peek()O(1)Just read top element
isEmpty()O(1)Check if top == -1 or head == nullptr
Search middleO(n)Must pop everything above it first
💡 Key Insight
All core stack operations are O(1) — constant time regardless of size. This is what makes stacks efficient.

Two Ways to Implement

Implementation 1 — Array

Use a fixed-size array + an integer top (starts at -1 when empty). Simple and fast.

✅ Fast & simple   ❌ Fixed max size

Implementation 2 — Linked List

Each element is a node with a pointer. head pointer = top of stack. Grows dynamically.

✅ Dynamic size   ❌ Extra pointer overhead

Stack with Array

// Building every function from scratch in C++
📌 Big Picture
We wrap an array and an integer top inside a class. top = -1 = empty. Push increments top, pop decrements it.

Mental Model — The Array + top Pointer

Think of the array as slots. top is an index (not a real pointer) that tracks the highest occupied slot.

Complete C++ Class

// ─── Stack Using Array ────────────────────────────────────────
#include <iostream>
using namespace std;

class Stack {
private:
    int arr[100];   // fixed-size array
    int top;         // index of current top element
    int capacity;    // maximum number of elements

public:
    // Constructor — top starts at -1 (empty)
    Stack(int size = 100) {
        top = -1;
        capacity = size;
    }

    // isEmpty: is top still at -1?
    bool isEmpty() {
        return top == -1;
    }

    // isFull: is top at the last valid index?
    bool isFull() {
        return top == capacity - 1;
    }

    // push: add element on top
    void push(int value) {
        if (isFull()) {
            cout << "Stack Overflow!\n";
            return;
        }
        top++;              // move top up
        arr[top] = value;   // store value
    }

    // pop: remove and return top element
    int pop() {
        if (isEmpty()) {
            cout << "Stack Underflow!\n";
            return -1;
        }
        int val = arr[top];  // save the value
        top--;               // move top down
        return val;
    }

    // peek: view top WITHOUT removing
    int peek() {
        if (isEmpty()) {
            cout << "Stack is empty!\n";
            return -1;
        }
        return arr[top];  // just return, don't change top
    }

    // display: print top to bottom
    void display() {
        if (isEmpty()) { cout << "Empty.\n"; return; }
        cout << "TOP: ";
        for (int i = top; i >= 0; i--) {
            cout << arr[i];
            if (i != 0) cout << " → ";
        }
        cout << endl;
    }

    // size: top+1 because top is an index
    int size() {
        return top + 1;
    }
};

Operation Deep Dives

PUSH — Step by Step

  • Check isFull() — if yes, print "Stack Overflow" and return early
  • Increment: top++
  • Store: arr[top] = value
⚠️ Stack Overflow
When top == capacity-1 and you try to push — that's Stack Overflow. Same term as recursive function overflowing the call stack!

POP — Step by Step

  • Check isEmpty() — if yes, print "Stack Underflow" and return -1
  • Save: int val = arr[top]
  • Decrement: top--
  • Return the saved value
💡 Pop Doesn't Erase
The old data is still in the array slot — it's just considered "garbage". The top pointer moved down, so that slot is invisible to the stack. Next push will overwrite it.

PEEK vs POP

peek()

return arr[top] — read the value, top stays unchanged.

pop()

Save value, then top-- — reads AND removes.

DISPLAY — The Loop

// Loop from top down to index 0
for (int i = top; i >= 0; i--) {
    cout << arr[i] << " ";
}
// If top=2, arr=[10,20,30] → prints: 30 20 10

Main Function Example

int main() {
    Stack s;
    s.push(10);   // top=0, arr=[10,...]
    s.push(20);   // top=1
    s.push(30);   // top=2
    s.display(); // TOP: 30 → 20 → 10
    cout << s.peek() << endl; // 30 (not removed)
    cout << s.pop()  << endl; // 30 removed, top=1
    cout << s.size() << endl; // 2
    return 0;
}

Stack with Linked List

// Dynamic — no fixed size limit
📌 Why Linked List?
With an array, size is fixed at compile time. With a linked list, every node is dynamically allocated — the stack grows/shrinks exactly as needed.

Node Structure & Visual

Each element is a node: data + a pointer to the node below it. The newest node is always at the head.

Newest element (30) is at HEAD. First element (10) has next = nullptr.

Complete C++ Implementation

#include <iostream>
using namespace std;

struct Node {
    int data;
    Node* next;
    Node(int val) : data(val), next(nullptr) {}
};

class Stack {
private:
    Node* head;   // the TOP of the stack
    int count;

public:
    Stack() : head(nullptr), count(0) {}

    // Destructor — MUST free all memory
    ~Stack() {
        while (!isEmpty()) pop();
    }

    bool isEmpty() {
        return head == nullptr;
    }

    // push: create node, link it, update head
    void push(int value) {
        Node* newNode = new Node(value); // 1. create
        newNode->next = head;              // 2. link to old top
        head = newNode;                    // 3. head = new top
        count++;
    }

    // pop: save head, move head, delete old node
    int pop() {
        if (isEmpty()) {
            cout << "Underflow!\n";
            return -1;
        }
        Node* temp = head;         // 1. save old top
        int val = temp->data;      // 2. save data
        head = head->next;         // 3. head moves down
        delete temp;              // 4. free memory!
        count--;
        return val;
    }

    int peek() {
        if (isEmpty()) { cout << "Empty!\n"; return -1; }
        return head->data;
    }

    void display() {
        Node* curr = head;
        cout << "TOP → ";
        while (curr) {
            cout << curr->data;
            if (curr->next) cout << " → ";
            curr = curr->next;
        }
        cout << " → NULL\n";
    }

    int size() { return count; }
};

Push — Pointer by Pointer

Pushing 40 onto [30 → 20 → 10]

  • Node* newNode = new Node(40) → data=40, next=nullptr
  • newNode->next = head → newNode now points to the node with 30
  • head = newNode → head now points to 40 (new top)

Result: head → 40 → 30 → 20 → 10 → NULL

Pop — Why delete is Critical

Popping from [40 → 30 → 20 → 10]

  • Node* temp = head → temp points to node 40
  • int val = temp->data → val = 40
  • head = head->next → head now points to node 30
  • delete temp → frees node 40 from heap memory
  • return val → returns 40
⚠️ Memory Leak
In C++, if you don't delete after popping, the node's memory is NEVER freed — it's a memory leak. This is why the destructor calls pop() until empty.

Balanced Parentheses

// A classic stack application

🎯 The Problem

Given a string with brackets ( ) [ ] { } — are they all properly matched and nested?

✅ (a + b) * [c - {d / e}]
❌ (a + b] * [c - {d / e}
❌ ((a + b)
❌ )a + b(

Why a Stack Works Here

When you see a closing bracket ), it must match the most recently seen opening bracket. "Most recently seen" = top of the stack. That's exactly LIFO!

The Algorithm

  • Create an empty stack. Loop through every character.
  • If it's an opener ( [ { → push it.
  • If it's a closer ) ] }:
    — Stack empty? → INVALID (no opener!)
    — Pop top. Does it match this closer? No → INVALID. Yes → continue.
  • After loop: stack empty → VALID. Not empty → INVALID (unclosed openers).

C++ Code

#include <iostream>
#include <stack>
#include <string>
using namespace std;

bool matches(char open, char close) {
    return (open == '(' && close == ')') ||
           (open == '[' && close == ']') ||
           (open == '{' && close == '}');
}

bool isBalanced(string expr) {
    stack<char> st;

    for (char ch : expr) {
        if (ch=='(' || ch=='[' || ch=='{') {
            st.push(ch);         // opener → push
        }
        else if (ch==')' || ch==']' || ch=='}') {
            if (st.empty()) return false;        // no opener!
            if (!matches(st.top(), ch)) return false; // mismatch!
            st.pop();
        }
        // other chars: skip
    }
    return st.empty();  // true = all brackets matched
}

int main() {
    cout << isBalanced("(a+b)*[c-{d}]") << endl;  // 1
    cout << isBalanced("(a+b]")           << endl;  // 0
    cout << isBalanced("((a)")            << endl;  // 0
}

Edge Cases — Don't Miss These!

More Closers Than Openers

Input: )abc( — first char is ')' and stack is empty → return false immediately.

Unclosed Openers

Input: ((a+b) — loop ends, stack still has '(' → stack.empty() = false → return false.

Mismatched Types

Input: (a+b] — see ']', pop '(' — matches('(', ']') = false → return false.

Empty String

Loop never runs. Stack is empty → return true. An empty string IS balanced.

Expression Evaluation

// Infix → Postfix conversion + stack-based evaluation

🎯 The Problem

We write math as 3 + 4 * 2 (infix). But evaluating this requires knowing precedence rules. Computers convert it to Postfix — a format that a stack can evaluate left-to-right without any precedence rules.

Three Notations

INFIX

A + B

Operator between operands. How humans write math. Needs parentheses for precedence.

PREFIX

+ A B

Operator before operands. Unambiguous but less common.

POSTFIX (RPN)

A B +

Operator after operands. Stacks evaluate this in one left-to-right pass. No parentheses needed!

Operator Precedence — Memorize This

3
Exponentiation (highest)
* /
2
Multiply / Divide
+ −
1
Add / Subtract
( )
—
Parentheses (never pop)

Part 1 — Infix to Postfix (Shunting-Yard)

We use an output string and an operator stack.

  • Loop through each token in the infix expression.
  • If it's an operand (letter/number) → add to output.
  • If it's ( → push to stack.
  • If it's ) → pop from stack to output until you hit (. Discard the (.
  • If it's an operator → pop stack to output while top has ≥ precedence. Then push current operator.
  • After loop → pop all remaining operators to output.

Example: A + B * C → Postfix

TokenActionStackOutput
Aoperand → output[ ]A
+stack empty → push[+]A
Boperand → output[+]A B
*prec(*)=2 > prec(+)=1 → push[+ *]A B
Coperand → output[+ *]A B C
endpop *, then +[ ]A B C * +

C++ — Infix to Postfix

#include <iostream>
#include <stack>
#include <string>
using namespace std;

int prec(char op) {
    if (op=='^') return 3;
    if (op=='*'||op=='/') return 2;
    if (op=='+'||op=='-') return 1;
    return 0;
}

string infixToPostfix(string expr) {
    stack<char> st;
    string output;

    for (char ch : expr) {
        if (isalnum(ch)) {
            output += ch;
        }
        else if (ch == '(') {
            st.push(ch);
        }
        else if (ch == ')') {
            while (!st.empty() && st.top() != '(') {
                output += st.top(); st.pop();
            }
            st.pop();  // discard '('
        }
        else {  // operator
            while (!st.empty() && st.top() != '(' &&
                   prec(st.top()) >= prec(ch)) {
                output += st.top(); st.pop();
            }
            st.push(ch);
        }
    }
    while (!st.empty()) { output += st.top(); st.pop(); }
    return output;
}

Part 2 — Evaluating a Postfix Expression

  • Scan left to right. Number → push.
  • Operator → pop B (right), pop A (left), compute A op B, push result.
  • Single remaining value in stack = the answer.
⚠️ Pop Order Matters!
For non-commutative ops (- and /): first pop = RIGHT operand (B), second pop = LEFT operand (A). Then compute A - B, not B - A!

Example: 5 3 2 * +

TokenActionStack
5push[5]
3push[5, 3]
2push[5, 3, 2]
*pop 2(B), pop 3(A), push 3*2=6[5, 6]
+pop 6(B), pop 5(A), push 5+6=11[11]

C++ — Postfix Evaluator

int evalPostfix(string postfix) {
    stack<int> st;
    for (char ch : postfix) {
        if (isdigit(ch)) {
            st.push(ch - '0');  // char to int
        } else {
            int B = st.top(); st.pop();  // RIGHT operand
            int A = st.top(); st.pop();  // LEFT operand
            switch(ch) {
                case '+': st.push(A+B); break;
                case '-': st.push(A-B); break;
                case '*': st.push(A*B); break;
                case '/': st.push(A/B); break;
            }
        }
    }
    return st.top();  // final answer
}

Summary Cheatsheet

Infix → Postfix Rules

• Operands → output
• ( → push
• ) → pop to output until (
• Operator → pop ≥ precedence ops first, then push
• End → pop all

Postfix Evaluation

• Number → push
• Operator → pop B (right), pop A (left), push A op B
• End → one value = answer
