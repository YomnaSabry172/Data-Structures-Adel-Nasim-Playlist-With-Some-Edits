Queue — Complete Interactive Guide

What is a queue?

Video #09 — Adil Naseem — Data Structures playlist

A queue is a data structure that follows the FIFO principle — First In, First Out. The element that enters first is the one that leaves first.

Real-life analogy

Think of a queue at a bank or cashier. The first person to stand in line is the first to be served. Nobody can cut in from the front. This is exactly how a queue works in code.

Two operations — that's all

Enqueue (insert)

Adding a new element always happens at the rear (back) of the queue.

Dequeue (remove)

Removing an element always happens from the front of the queue.

The problem with a simple linear array

Imagine an array of size 5. You enqueue 5 elements, then dequeue 3. The front pointer moves forward — but those 3 freed cells at the start are wasted forever. Even though there is space, the queue reports "full" because rear reached the last index. This is the false overflow problem.

Problem: after dequeues, the front cells can never be reused in a simple array queue. This is called "false overflow" — the queue thinks it is full when it is not.

// After 5 enqueues and 3 dequeues:

wasted [0]
wasted [1]
wasted [2]
40 front [3]
50 rear [4]

// rear == 4 == SIZE-1. Cannot enqueue even though 3 slots are free!

The solution — circular queue

A circular queue treats the array as a circle. When rear reaches the last index, it wraps back to index 0 — reusing freed cells. The trick is one formula: rear = (rear + 1) % SIZE

Simple array queue

front and rear only move forward.
Wastes space after dequeues.
Gets a false overflow.

Circular array queue

rear wraps around using % SIZE.
No wasted space.
Solves false overflow completely.

Quick summary

1
Queue = FIFO. First element added is the first removed.
2
Enqueue always adds at the rear. Dequeue always removes from the front.
3
Simple array queue has a false overflow problem — wasted cells after dequeues.
4
Circular queue fixes this by wrapping rear back to index 0 using the modulo operator.

Circular array queue

Video #10 — Queue implementation using circular array

We store elements in a fixed-size array. Two pointers — front and rear — track where the queue starts and ends. The magic is the modulo operator that makes them wrap around.

isEmpty condition

front == -1
Both start at -1. When the queue is empty, front stays at -1.

isFull condition

(rear + 1) % SIZE == front
The next position of rear would collide with front.

The key wrap-around formula

rear = (rear + 1) % SIZE
When rear is at index 4 (last) and SIZE is 5: (4 + 1) % 5 = 0. Rear jumps back to index 0. That is the entire circular trick.

Complete C++ implementation

circular_queue.cpp 
#include <iostream>
using namespace std;

const int SIZE = 5;  // max elements we can hold is SIZE-1 = 4
                      // (one slot always kept empty to detect full)

class CircularQueue {
    int arr[SIZE];
    int front;
    int rear;

public:
    CircularQueue() {
        front = -1;   // -1 = queue is empty
        rear  = -1;
    }

    // ─── isEmpty ────────────────────────────────────────
    bool isEmpty() {
        return front == -1;
    }

    // ─── isFull ─────────────────────────────────────────
    bool isFull() {
        // if rear's NEXT position equals front → no room left
        return (rear + 1) % SIZE == front;
    }

    // ─── enqueue ────────────────────────────────────────
    void enqueue(int value) {
        if (isFull()) {
            cout << "Queue is FULL! Cannot enqueue." << endl;
            return;
        }
        if (isEmpty()) {
            front = 0;   // very first element: set both to 0
            rear  = 0;
        } else {
            rear = (rear + 1) % SIZE;  // wrap rear forward
        }
        arr[rear] = value;
        cout << "Enqueued: " << value << endl;
    }

    // ─── dequeue ────────────────────────────────────────
    void dequeue() {
        if (isEmpty()) {
            cout << "Queue is EMPTY! Cannot dequeue." << endl;
            return;
        }
        cout << "Dequeued: " << arr[front] << endl;

        if (front == rear) {
            // that was the last element — reset to empty
            front = -1;
            rear  = -1;
        } else {
            front = (front + 1) % SIZE;  // wrap front forward
        }
    }

    // ─── display ────────────────────────────────────────
    void display() {
        if (isEmpty()) { cout << "Empty." << endl; return; }
        cout << "Queue: ";
        int i = front;
        while (true) {
            cout << arr[i] << " ";
            if (i == rear) break;
            i = (i + 1) % SIZE;
        }
        cout << endl;
    }
};

int main() {
    CircularQueue q;
    q.enqueue(10);   // front=0 rear=0  → [10, _, _, _, _]
    q.enqueue(20);   // front=0 rear=1  → [10,20, _, _, _]
    q.enqueue(30);   // front=0 rear=2  → [10,20,30, _, _]
    q.dequeue();    // removes 10, front=1
    q.enqueue(40);   // rear=3
    q.display();    // prints: 20 30 40
    return 0;
}

Step-by-step walkthrough

1
Start: front = -1, rear = -1. Both are -1, meaning the queue is empty.
2
First enqueue: front = 0, rear = 0. The very first element sets both pointers to 0.
3
Each enqueue after: rear = (rear + 1) % SIZE. If rear was at index 4, it wraps to 0.
4
Dequeue: read arr[front], then front = (front + 1) % SIZE. If it was the last element, reset both to -1.
5
isFull: (rear + 1) % SIZE == front. One slot is intentionally left empty so we can tell full from empty.

Queue using linked list

Video #11 — No fixed size. Memory grows dynamically.

Instead of a fixed array, each element is a Node object in memory. Nodes are connected via pointers. We keep two pointers — front and rear — to the start and end of the chain.

Why linked list?

The array queue has a fixed maximum size set at compile time. A linked list queue has no size limit — it grows and shrinks as needed. Memory is allocated only when required.

The node structure

Every node has two fields: data — the value stored, and next — a pointer to the next node. The last node's next is nullptr.

Complete C++ implementation

linked_queue.cpp 
#include <iostream>
using namespace std;

// ─── Node: one link in the chain ────────────────────
struct Node {
    int   data;    // the value stored in this node
    Node* next;    // pointer to the next node (nullptr if last)

    Node(int val) {
        data = val;
        next = nullptr;
    }
};

// ─── LinkedQueue ────────────────────────────────────
class LinkedQueue {
    Node* front;   // points to first node (dequeue from here)
    Node* rear;    // points to last node  (enqueue here)

public:
    LinkedQueue() {
        front = nullptr;
        rear  = nullptr;
    }

    // ─── isEmpty ────────────────────────────────────────
    bool isEmpty() {
        return front == nullptr;
    }

    // ─── enqueue: create a new node, attach at rear ─────
    void enqueue(int value) {
        Node* newNode = new Node(value); // allocate on heap

        if (isEmpty()) {
            // first node: both front AND rear point to it
            front = newNode;
            rear  = newNode;
        } else {
            rear->next = newNode;  // old rear points forward to new
            rear       = newNode;  // rear itself moves to new node
        }
        cout << "Enqueued: " << value << endl;
    }

    // ─── dequeue: remove from front ─────────────────────
    void dequeue() {
        if (isEmpty()) {
            cout << "Queue is EMPTY!" << endl;
            return;
        }
        Node* temp = front;           // save to delete later
        cout << "Dequeued: " << front->data << endl;
        front = front->next;          // move front to next node
        if (front == nullptr) {
            rear = nullptr;          // queue is now empty
        }
        delete temp;                  // free the removed node's memory!
    }

    // ─── display: walk chain front → rear ───────────────
    void display() {
        if (isEmpty()) { cout << "Empty." << endl; return; }
        Node* curr = front;
        cout << "Queue: ";
        while (curr != nullptr) {
            cout << curr->data;
            if (curr->next) cout << " → ";
            curr = curr->next;
        }
        cout << " → nullptr" << endl;
    }

    // ─── destructor: free all remaining nodes ───────────
    ~LinkedQueue() {
        while (!isEmpty()) dequeue();
    }
};

int main() {
    LinkedQueue q;
    q.enqueue(10);  // front → [10|*] ← rear
    q.enqueue(20);  // front → [10|*] → [20|*] ← rear
    q.enqueue(30);  // front → [10|*] → [20|*] → [30|*] ← rear
    q.display();   // 10 → 20 → 30 → nullptr
    q.dequeue();   // removes 10; front now at [20]
    q.display();   // 20 → 30 → nullptr
    return 0;
}

Step-by-step walkthrough

1
Start: front = nullptr, rear = nullptr. The queue is empty.
2
Enqueue (first node): new Node(value) creates a node on the heap. Both front and rear point to it.
3
Enqueue (subsequent): rear->next = newNode links old rear to new, then rear = newNode moves rear.
4
Dequeue: save front in temp, move front = front->next, then delete temp to free memory.
5
Last element dequeued: front becomes nullptr. Also set rear = nullptr — queue is empty again.

Array queue vs linked list queue — comparison

Feature Circular array queue Linked list queue
Size Fixed at compile time Dynamic — grows as needed
Memory per element Just the data (int) Data + pointer (extra 8 bytes)
Can overflow? Yes — when full No — unlimited
Speed Faster (cache-friendly array) Slightly slower (heap allocations)
Memory leaks Not possible (stack memory) Possible if you forget delete
Best used when Size is known in advance Size is unknown / varies a lot
